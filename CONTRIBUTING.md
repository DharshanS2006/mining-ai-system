# Contributing to Mining AI System

Thanks for your interest in contributing! We welcome bug fixes, features, docs, and improvements.

## How to contribute

1. Fork the repository.
2. Create a branch (`git checkout -b feature/your-feature`).
3. Make your changes.
4. Add/Update tests.
5. Run tests: `pytest -q`.
6. Commit: `git commit -m "Add ..."`.
7. Push: `git push origin feature/your-feature`.
8. Open a pull request.

## Code style

- Python: PEP 8
- Use descriptive variable names
- Keep functions small and focused
- Add comments for non-obvious logic

## Branch strategy

- `main`: production-ready
- `develop`: development branch (if used)
- feature branches: `feature/*`
- bugfix branches: `bugfix/*`
- release branches: `release/*`

## Review process

- A minimum of one approving review required
- Address requested changes in follow-up commits
- CI must pass before merge

## Reporting issues

1. Open a new issue.
2. Use the bug report template.
3. Include environment, reproduction steps, and expected behavior.
